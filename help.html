1 — Overview

The Efficy Labels Editor is a browser-based tool for editing .txt localization files used by Efficy CRM. It runs entirely in your browser — there is no server, no install, and no internet connection required (except for the optional AI translation feature).

Open index.html directly in your browser, load a labels file, edit it, and download the result.

2 — File format

Label files are UTF-8 encoded .txt files saved with a BOM (\uFEFF). Each line is one of:

Line typeExampleDescription
Header ==> Languages: EN FR NL Declares the language columns; tab-separated after the colon.
Comment // Section: Contacts Starts with //. Visible and editable in the table.
Data BTN_SAVE Save Enregistrer Opslaan Tab-separated: key, then one value per language.
Empty (blank line) Preserved as-is for readability; not shown in the table.
Tip: The tool saves files with a UTF-8 BOM so they open correctly in Excel and legacy tools that expect it.

3 — Loading a file

Click 📁 Load File and pick a .txt labels file from your computer. The file is read entirely in the browser — nothing is uploaded anywhere.

Once loaded, the filename is shown in the toolbar and all editing controls become active.

4 — Editing labels

Every cell in the table is directly editable:

  • Key column — the label identifier (e.g. BTN_SAVE). Changes are reflected immediately in the in-memory data.
  • Language columns — one column per language declared in the header. Edit any value in-place.
  • Comment rows — shown with a yellow background. The whole row is one editable text field.
Tip: Changes are held in memory only. Use Save & Download or Backup to persist them.

5 — Searching

Use the search bar below the toolbar to filter the table. Two modes are supported:

Fuzzy match (default)

Characters must appear in order but not necessarily consecutively. For example, svcont matches SAVE_CONTACT.

Exact match

Wrap your term in double quotes: "save contact". The table will only show rows where the search term appears as a contiguous substring (case-insensitive).

The search looks across the key, all language values, and comment text simultaneously. The row counter in the toolbar updates to show matching / total.

Tip: Clearing the search box instantly restores the full list.

6 — Pagination

The pagination bar sits between the search bar and the table. It shows:

  • X / Total — rows shown on the current page vs. total matching rows.
  • « ‹ › » — navigate to first, previous, next, last page. Buttons grey out at boundaries.
  • Rows per page — choose 10, 25, 50, 100, or 200. Default is 25. Changing this resets to page 1.

7 — Adding rows & comments

➕ Add Row

Appends a new empty data row at the end of the file. The table automatically jumps to the last page so the new row is visible. Fill in the key and values, then save.

💬 Add Comment

Appends a new comment line (// ) at the end. Useful for grouping related labels by section. The comment is editable directly in the table.

8 — Saving & backup

💾 Save & Download

Downloads the current state of the file under its original filename. The file is serialized back to the tab-separated format with a UTF-8 BOM.

🗄️ Backup

Downloads a copy with an auto-generated timestamped name, e.g. efficy_labels_bkp_20260325143022.txt. Use this before making large edits.

Important: The browser does not auto-save. If you close the tab without downloading, your changes are lost.

9 — AI translation (optional)

The ✨ AI Translate ✓ button lets you translate individual empty cells using Claude (Anthropic's AI). This feature is entirely optional and requires an Anthropic API key.

Setting up your API key

Click ✨ AI Translate to open the key management modal. You have two ways to get a key:

  1. Claude Code CLI shortcut — if you have Claude Code installed, your key is already saved locally at:
    %USERPROFILE%\.claude\config.json → copy the value of primaryApiKey.
  2. Anthropic console — go to console.anthropic.com → Settings → API Keys → Create Key.

The key is saved in your browser's localStorage and is never sent anywhere except directly to api.anthropic.com.

Using translate buttons

Once a key is configured, a ✨ button appears next to every empty language cell. Click it to auto-translate based on the label key and any existing translations for context. The model used is claude-haiku-4-5 (fast and cost-effective).

If you manually clear a cell that previously had a value, the ✨ button will appear automatically.

Removing the key

Re-open the modal and click Remove key. The key is deleted from localStorage immediately.

10 — DeepL

The 🌐 DeepL button simply opens deepl.com/translator in a new tab as a quick reference for manual translations. No data is sent automatically.
